## Current status

- Bug 41 (`interrupted generation`) was taken through several iterations on branch `fix/ghostwriter-bug-41-interrupted-generation`.
- The immediate stop/resume race was fixed: stopping generation now returns the UI to idle and the next send no longer replays stale chunks from the previous request.

- Conclusion: this is not just a streaming-state bug. It needs a deeper look at the follow-up prompt strategy and/or continuation context construction after a stop.
